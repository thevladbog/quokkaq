# Цифровое табло (экран юнита)

В QuokkaQ **один пункт обслуживания (unit)** сопоставлен **одному публичному экрану** (`/screen/...`). Это не сеть DSP на сотни дисплеев: ценность в **связке с очередью** (талоны, ETA) и в **сезонных / срочных сообщениях**.

## Плейлисты и слайды

- **Плейлист** — упорядоченный список **материалов** (картинки/видео) с длительностью по слайду.
- Необязательные **«С даты / По дату»** (YYYY-MM-DD) на **слайде** ограничивают период показа. Даты считаются в **таймзоне юнита** (как и «сегодня» в очереди).

## Расписания (время суток и календарь)

- **Расписание** сопоставляет плейлист с **днями недели** (1 = пн … 7 = вс) и **интервалом времени** (в т.ч. «через полночь»).
- Необязательные **«С даты / По дату»** на расписании ограничивают **календарный** период действия правила (включительно) в **таймзоне юнита**. Пусто = без границы.
- При нескольких подходящих слотах выигрывает **больший приоритет**; в админке недельный вид подсвечивает **пересечения** и слоты **вне сегодняшнего календаря**, если заданы даты.

## Плейлист по умолчанию

- Один плейлист можно отметить как **по умолчанию** — запасной вариант, если нет подходящего расписания на текущее время (или все календарные окна не включают «сегодня»).

## Внешние фиды

- RSS, погода, произвольный URL — периодический опрос. Во вкладке **Состояние** видны последний успех и серия ошибок.

## Вкладка «Состояние»

- Сводка: откуда берётся активный плейлист, пусто ли после фильтра по датам, есть ли **плейлист по умолчанию**, здоровье **внешних фидов** (последний успех / подряд идущие сбои).

## Объявления

- **В макете (баннер)** — полоса в раскладке.
- **Полноэкранно** — оверлей с высокой заметностью (учитывается **снижение анимации** в системе). В админке есть **превью** до публикации (текст не публикуется, пока не нажмёте «Add»).
