# 1С:УНФ — обмен с QuokkaQ (CommerceML)

Краткий чек-лист для настройки узла обмена в **1С:Управление нашей фирмой** (ред. 3.x) и сопоставления данных с сайтом (SaaS). Подробности см. в методической документации 1С по интеграции с сайтами (CommerceML 2.05+, для оплат — **2.10**).

## Подключение HTTP

- В QuokkaQ: **Интеграции → 1С УНФ**, включить обмен, задать логин/пароль, скопировать URL эндпоинта (`API_PUBLIC_URL` / `PUBLIC_APP_URL` + `/commerceml/exchange`).
- В мастере УНФ: тип сайта **«Другой»**, обмен по **HTTP**, указать URL, логин и пароль, проверить соединение.

## Новые контрагенты (тенанты)

- В заказе из QuokkaQ в блоке контрагента передаётся **стабильный идентификатор с сайта** (`Ид` = UUID организации в QuokkaQ), наименование и при наличии **ИНН**, **КПП**, полное наименование из карточки компании / снимка на счёте.
- В узле обмена на шаге загрузки заказов задайте способ загрузки контрагентов, например **«Искать по реквизитам, создавать, если не найден»**, и отметьте поля поиска (уникальный идентификатор, ИНН, наименование, контакты и т.д. — по политике учёта).
- После того как в справочнике 1С появится «правильный» контрагент, при необходимости можно вручную задать **onecCounterpartyGuid** в админке платформы (сопоставление с `Ид` из 1С) — это необязательно для первой выгрузки.

## Онлайн-оплаты (эквайринг)

- Версия схемы CommerceML для передачи оплат с сайта — **2.10**.
- В мастере УНФ на шаге **«Оплата»**: включить обмен оплатами/отгрузками при необходимости; **сопоставить платёжные системы на сайте** с эквайринговыми терминалами / кассами в 1С.
- В QuokkaQ в настройках 1С можно задать **«платёжная система на сайте»** (`sitePaymentSystemName`) — она попадает в XML как `НаименованиеПлатежнойСистемы` для сопоставления с узлом.
- Для загрузки онлайн-оплат в 1С как **«Операция по платежным картам»** операции должны быть с видом **Эквайринг** и с заполненными **номером платёжного документа** и **датой оплаты** (QuokkaQ передаёт их для оплат YooKassa/Stripe).

## Статусы счёта

Здесь два независимых слоя — не путайте мастер УНФ с настройками QuokkaQ.

- **Сторона 1С (как в методичке по узлу обмена с сайтом).** Сопоставление состояний заказа на сайте и документов в 1С задаётся **в мастере настройки узла обмена в УНФ** (соответствие «наших» и «их» статусов в терминах обмена). Подробности — в методической документации 1С по интеграции с сайтами для вашей конфигурации.
- **Сторона QuokkaQ (технический слой при `sale/import`).** Когда 1С передаёт обратно в сообщении CommerceML **текст статуса** заказа/документа, приложению нужно привести его к статусу счёта в SaaS (`paid` / `void` / `uncollectible`). Опциональный **JSON маппинг в настройках интеграции 1С в QuokkaQ** задаёт дополнительные правила (подстрока или точное совпадение по пришедшей строке) — это **не замена** настройкам узла в УНФ, а донастройка интерпретации входящего текста на сервере QuokkaQ.
- Если JSON не задан, используются **встроенные эвристики** (например, по подстрокам вроде «оплачен» / `paid` для перевода счёта в оплаченный).
