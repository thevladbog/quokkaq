# Киоск: дизайн и UX — анализ и план улучшений

## Текущий дизайн: что есть

### Сетка услуг
Фиксированная матрица **8×8 = 64 ячейки**. Каждая услуга размещается вручную через поля `gridRow`, `gridCol`, `gridRowSpan`, `gridColSpan`. Тайлы растягиваются на произвольное количество ячеек. Зазоры: 6px (mobile) → 8px (sm) → 12px (md). Gap между ячейками:  `gap-1.5` → `gap-2` → `gap-3`.

### Тайл услуги
- Форма: `rounded-3xl` (24px)
- Фон: кастомный `backgroundColor` из конфига, или дефолтный warm cream `oklch(0.98 0.01 75)`
- Тень: двухслойная, коричневые тоны `rgba(29,27,25, 0.08/0.06)`
- Типографика: fluid sizing через `clamp()` + container queries `@container/kiosk-tile`
  - Заголовок: `clamp(1.125rem, calc(4.2cqw + 1.15vmin), 2.5rem)`
  - Описание: `clamp(0.8125rem, calc(2.45cqw + 0.65vmin), 1.125rem)`
- Touch feedback: `active:scale-[0.99]` — почти незаметно
- Если есть изображение: двухколоночный layout (42% изображение / 58% текст) при ширине тайла ≥15rem, иначе стек

### Топ-бар
- Высота: 4.5rem (mobile) → 5rem (sm+)
- Фон: `headerColor` (дефолт `#fff9f4`)
- Содержимое: логотип + название юнита / панель доступности / пре-регистрация / смена языка / часы

### Модал подтверждения (успех)
- Ширина: `w-[320px] sm:w-[420px]` — ограничена как у мобильного диалога
- Номер очереди: `text-7xl leading-none font-bold` (~72px)
- QR-код: 180px
- Авто-закрытие: **5 секунд**

### Палитра
- Тёплые нейтралы: Header `#fff9f4`, Body `#fef8f3`, Grid `#f2ebe6`
- Текст: `oklch(0.3 0.035 55)` — тёмно-коричневый
- Граница: `oklch(0.9 0.012 75)` — очень светлая
- High-contrast mode: принудительный чёрный фон + белый текст, amber фокус-кольцо

### Доступность (уже реализовано частично)
- Панель в TopBar: 3 шага размера шрифта (1×/1.12×/1.22×), высококонтрастный режим, TTS, выбор аудиоустройства
- Focus rings: 3px amber, outline-offset 3px
- `.kiosk-touch-min`: min 48px для touch-целей
- `prefers-reduced-motion`: анимации отключаются

### Чего нет
- Attract/idle screen (экран ожидания) — при бездействии просто висит сетка услуг
- Анимации подтверждения (нет checkmark draw, нет входа номера)
- Визуального различия между «листовой» услугой (→ талон) и «ветвью» (→ подуслуги)
- Иконок по умолчанию для услуг без `imageUrl`
- Автоматического расположения тайлов (только ручная раскладка через gridRow/Col)

## Анализ vs мировые практики

| Аспект | Мировой стандарт | Текущий QuokkaQ | Оценка |
|---|---|---|---|
| Сетка | 2×2–3×3, автоматическая, ≤9 услуг на экран | 8×8 ручная, до 64 ячеек | ⚠️ Мощно, но сложно в настройке |
| Attract screen | Есть у всех: анимация + «Touch to begin» | Отсутствует | ❌ |
| Touch feedback | scale 0.93–0.96 + цвет | scale 0.99 — незаметно | ⚠️ |
| Confirmation modal | Full-screen или большой, номер 120–180px | 420px wide, номер 72px | ⚠️ |
| Авто-закрытие | 10–15 сек с отсчётом | 5 сек — очень быстро | ⚠️ |
| Иконка + текст | Стандарт у Qmatic, SEDCO, Moviik | Только текст без imageUrl | ❌ |
| Лист vs ветвь | Стрелка/шеврон на «ветвях» | Нет визуального отличия | ❌ |
| Тайм-аут с attract | 30–60 сек → плавный переход | Только warning modal без перехода | ⚠️ |
| Контраст текста | 7:1 для кiosков (сверх WCAG AA) | ~6–7:1 дефолт, проверяется | ✅ |
| Fluid typography | clamp + container queries | ✅ реализовано | ✅ |
| Numeric pad | ≥80px кнопки | h-[3.25rem] ≈ 52px | ⚠️ |
| High contrast mode | Обязателен | ✅ реализован | ✅ |
| Font scaling | Обязателен | ✅ 3 шага | ✅ |
| Размер касания | ≥20мм физических (≈80–116px на 1080p) | min-h-12 = 48px (недостаточно на 1080p) | ⚠️ |

## Рекомендации по улучшениям

### Дизайн 1 — Attract / Idle screen (экран ожидания)

**Проблема:** Самый распространённый паттерн в мировых QMS-системах полностью отсутствует. Без него киоск не привлекает проходящих посетителей и выглядит как «сломанный» после бездействия.

**Что нужно:**
- Через 60 сек бездействия — плавный fade-in к attract screen поверх сетки услуг
- Contain: логотип/брендинг, анимированный или видео-фон, крупный CTA «Нажмите для начала» с pulsing-анимацией
- Текущее время (reassures user, что терминал живой)
- Опционально: «Сейчас в очереди X человек» / «Примерное ожидание Y мин»
- Тап/клик по любому месту → немедленный возврат к сетке услуг
- Если `welcomeTitle`/`welcomeSubtitle` настроены — использовать их

**Реализация:**
- Компонент `KioskAttractScreen` (новый), рендерится через `AnimatePresence` с `framer-motion`
- Состояние `isIdle: boolean` в хуке `useIdleTimer(60_000)`
- `useIdleTimer` уже нужен для таймаута сессии (фича 1.4 из функционального плана) — объединить

**Визуальная схема:**
```
┌─────────────────────────────────────────┐
│           [Logo]                        │
│                                         │
│      Добро пожаловать!                  │  ← welcomeTitle
│   Нажмите, чтобы начать                 │  ← pulsing button
│                                         │
│      В очереди сейчас: 3 чел.           │  ← live WebSocket
│                                         │
│                          15:42  пт      │
└─────────────────────────────────────────┘
```

### Дизайн 2 — Автоматическая раскладка тайлов (Smart Auto-layout)

**Проблема:** 8×8-сетка с ручным `gridRow/Col/RowSpan/ColSpan` — мощный инструмент, но для большинства клиентов это избыточная сложность. Если администратор не проставит координаты, услуги не отобразятся. Мировая практика: системы сами раскладывают услуги оптимально.

**Решение: два режима сетки параллельно**

**Режим A — Manual (текущий):** Сохранить как есть. Для клиентов с кастомными макетами (нестандартные span, featured-тайлы).

**Режим B — Auto-layout (новый):** Если у услуги нет заданных gridRow/Col (или включён флаг `autoLayout: true` в KioskConfig), система сама размещает тайлы по оптимальной сетке:

| Кол-во услуг | Авто-сетка | Размер тайла |
|---|---|---|
| 1 | 1×1 | Full-screen content area |
| 2 | 1×2 | Half-width, full-height |
| 3 | 2×2 (3 тайла + 1 пустой) | Even |
| 4 | 2×2 | Equal |
| 5–6 | 2×3 | Equal |
| 7–9 | 3×3 | Equal |
| 10–12 | 3×4 | Smaller |
| 13+ | 3×3 + пагинация (← →) | Equal + page dots |

**Пагинация вместо скролла** — никогда не scroll на кioске; явные кнопки «← Назад» / «Далее →» + счётчик «стр. 1 из 2».

**Реализация:**
- Утилита `computeAutoLayout(services[])` → возвращает массив с `gridRow/Col/RowSpan/ColSpan`
- Логика в `apps/frontend/src/lib/service-grid.ts` (уже существует)
- KioskConfig: флаг `serviceGridLayout: 'manual' | 'auto'` (дефолт `'auto'` для новых юнитов)

### Дизайн 3 — Визуальное различие: лист vs ветвь

**Проблема:** Тайл услуги-категории (ведёт в подуслуги) и тайл услуги-листа (сразу создаёт талон) выглядят одинаково. Пользователь не знает, ожидать ли ещё один экран.

**Решение:**
- Ветвь (есть подуслуги): добавить шеврон `›` или `→` в правый нижний угол тайла
- Листовая услуга: добавить иконку «талон» или «✓» в правый нижний угол
- Цвет индикатора: `text-kiosk-ink/40` (приглушённый, не перебивает название)
- Опционально: лёгкий gradient на нижнем крае тайла-ветви для ощущения «глубины»

```
┌──────────────────┐     ┌──────────────────┐
│                  │     │                  │
│   Паспортный     │     │   Консультация   │
│   стол           │     │                  │
│               ›  │     │              ✓   │
└──────────────────┘     └──────────────────┘
    (ветвь)                   (лист)
```

**Реализация:** `KioskServiceTile` → принимает prop `isLeaf: boolean`, рендерит соответствующий indicator.

---

### Дизайн 4 — Иконки по умолчанию для услуг

**Проблема:** Без `imageUrl` тайл — чистый текст. Мировой стандарт: иконка + текст всегда.

**Решение:**
- Библиотека дефолтных иконок по категориям (из Lucide, уже используется):
  ```
  "health" / "medical"  → Stethoscope
  "document" / "docs"   → FileText
  "payment" / "finance" → CreditCard
  "queue" / "ticket"    → Ticket
  "consult"             → MessageCircle
  "info"                → Info
  "passport"            → BookUser
  "social"              → Users
  default               → ChevronRight (нейтральный)
  ```
- Маппинг: поле `iconKey?: string` в модели услуги (новое необязательное поле)
- Если `imageUrl` не задан и `iconKey` не задан → показывать первую букву названия услуги в цветном круге (как аватар) — `bg-primary/15 text-primary font-bold text-2xl`
- Emoji в `imageUrl`: если строка — emoji-символ (unicode), рендерить как `text-5xl`

### Дизайн 5 — Полноэкранный confirmation / success state

**Проблема:** Модал 420px на 21" экране кioска выглядит маленьким и неубедительным. Номер очереди 72px — читается с 50 см, но не с 1.5 м. Авто-закрытие через 5 сек — слишком быстро.

**Решение — два варианта:**

**Вариант A (рекомендуется): Full-screen overlay**
- Вместо Dialog → `<AnimatePresence>` с фиксированным full-screen div
- Три зоны:
  ```
  ┌─────────────────────────────────────────┐
  │  [Logo]          [Checkmark ✓ 80px]     │  ← 20% экрана
  ├─────────────────────────────────────────┤
  │                                         │
  │         Консультация                    │  ← имя услуги
  │                                         │
  │             047                         │  ← номер, 140–180px
  │                                         │
  │       Впереди вас: 3 человека           │
  │       Примерное ожидание: 12 мин        │  ← 60% экрана
  │                                         │
  ├─────────────────────────────────────────┤
  │  [QR-код 200px]  Отсканируйте для       │
  │                  отслеживания           │
  │                  Закрытие через 12 сек  │  ← 20% экрана
  └─────────────────────────────────────────┘
  ```

**Вариант B (консервативный): Расширить модал**
- `w-full max-w-[600px]` на sm+
- Номер: `text-8xl sm:text-9xl` (96–128px)
- QR: 220px

**Авто-закрытие:** изменить с 5 до **12 секунд** (достаточно прочитать и сделать фото/скан QR)

**Анимация подтверждения:**
1. Checkmark: stroke draw animation (500ms, ease-out)
2. Номер: fade + translateY(+8px → 0) (300ms, delay 200ms)
3. ETA/позиция: fade-in (200ms, delay 400ms)
4. QR-код: fade-in (200ms, delay 500ms)

Все анимации отключаются при `prefers-reduced-motion`.

### Дизайн 6 — Touch-feedback на тайлах

**Проблема:** `active:scale-[0.99]` — изменение на 1% практически незаметно пальцем на сенсорном экране.

**Решение:**
- Увеличить до `active:scale-[0.96]` (4% — воспринимаемо, не агрессивно)
- Добавить изменение фона при active: `active:brightness-[0.93]`
- На тайлах с кастомным `backgroundColor`: darken на 7% при нажатии

**Также:** добавить `transition-[transform,filter]` (сейчас только `transform,box-shadow`).

---

### Дизайн 7 — Размеры кнопок в numpad (телефон/код)

**Проблема:** Кнопки цифровой клавиатуры `h-[3.25rem]` (52px) — меньше рекомендуемого физического минимума 20мм ≈ 80–116px на 1080p-экране.

**Решение:**
- `h-[4.5rem] sm:h-[5rem]` (72px → 80px)
- Шрифт цифр: `text-3xl sm:text-4xl` (уже так), проверить что не режется
- Увеличить grid gap: `gap-2 sm:gap-3`
- Модалы `KioskPhoneIdentificationModal` и `PreRegRedemptionModal`

---

### Дизайн 8 — Плавный переход к attract вместо «зависания»

**Проблема:** После таймаута сессии показывается warning modal, затем сброс на сетку. Нет плавного перехода к attract screen. Выглядит как «система зависла».

**Решение:**

**Состояния и переходы:**
```
ACTIVE ──(60s бездействия)──→ WARNING (15s обратный отсчёт)
  ↑                               │ (тап = возврат)
  │                               │ (истёк = сброс)
  └──(тап)──← ATTRACT ←──────────┘
                │ (60s без тапа)
                ↓
            SCREENSAVER (если нужен, опционально)
```

- `ACTIVE → WARNING`: bottom-bar оповещение (не блокирующий модал!) — «Сессия завершится через 15 сек. Нажмите для продолжения.»
- `WARNING → ATTRACT`: fade-out сетки → fade-in attract screen (400ms cross-fade)
- `ATTRACT → ACTIVE`: немедленно при любом касании

**Почему bottom-bar, не модал:** блокирующий модал мешает пользователю, который ещё взаимодействует с периферийным вниманием; bottom-bar информирует, не блокируя.

### Дизайн 9 — Тема для яркого/тёмного окружения

**Проблема:** Текущая тёплая нейтральная тема (#fff9f4) хорошо работает в офисе/больнице, но может выцветать в ярко освещённых торговых центрах или медиаценностных пространствах.

**Решение:** Добавить в KioskConfig выбор **базовой темы:**

| Тема | Background | Для окружения |
|---|---|---|
| `warm-light` (текущий) | #fff9f4 / #fef8f3 | Офис, больница, МФЦ |
| `cool-light` | #f8faff / #f0f4ff | Банк, государственный сектор |
| `dark` | #0f0f0f / #1a1a1a | Dim retail, premium, ночная смена |
| `high-contrast` | #000000 / #111111 | Outdoors, яркий свет, ADA |

- Базовая тема задаётся в KioskConfig: `baseTheme: 'warm-light' | 'cool-light' | 'dark' | 'high-contrast'`
- Поверх неё работают кастомные цвета (`isCustomColorsEnabled`)
- `dark` тема: автоматически переключает иконки, тени, border-цвета

### Дизайн 10 — Индикатор очереди на attract screen и сетке

**Проблема:** Посетитель не знает, стоит ли занимать очередь прямо сейчас — нет визуального сигнала о загруженности.

**Решение:**
- На attract screen: «Сейчас в очереди: N человек. Примерное ожидание: X мин.» (из WebSocket `queue.snapshot`)
- Опционально: цветовой индикатор загруженности (зелёный/жёлтый/красный) в топ-баре
- KioskConfig: `showQueueDepthOnAttract: boolean` (дефолт `true`)

**Реализовано (фаза B):** В `KioskConfig` зафиксированы `kioskAttractInactivityMode` (`session_then_attract` | `attract_only` | `off`, по умолчанию `session_then_attract`), `showAttractAfterSessionEnd` (по умолчанию `true`), `attractIdleSec` (10–600, по умолчанию 60, для `attract_only`) и `showQueueDepthOnAttract` (по умолчанию `true`). Сессия: нижняя полоса предупреждения вместо модалки; при `session_then_attract` после отсчёта — attract при необходимости. Нижняя полоса: `NEXT_PUBLIC_APP_VERSION` + бейдж сети/заморозки. **Attract+signage:** экран в шапка (лого + время) / центр / CTA; контент слайдера — тот же источник, что публичный дисплей очереди: активный плейлист `getActivePlaylist` иначе `adScreen.activeMaterialIds` + `ContentPlayer`; при отсутствии слайдов — в центре ETA (если включено) и кнопка; тап на весь экран — только без рекламы, со слайдами — выход по кнопке.

## Что НЕ менять

**Оставить как есть:**
- Fluid typography через `clamp()` + container queries — это современный и правильный подход, редко встречается у конкурентов
- Warm neutral палитра дефолт — подходит для большинства целевых сегментов (здравоохранение, МФЦ, сервис)
- Rounded-3xl тайлы — мягкий, дружелюбный вид соответствует позиционированию «the friendliest QMS»
- Двухслойные тени с коричневым тоном — дают глубину без резкости
- Существующая система 8×8 manual grid — мощный инструмент для кастомных раскладок, оставить как режим `manual`
- High contrast mode + TTS + font scaling — уже реализованы, нужно лишь тестирование
