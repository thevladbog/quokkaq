# Лендинг (marketing): анализ и план улучшений

## Текущее состояние

Сайт состоит из 11 секций:
1. Sticky-навигация (лого, анкоры, CTA «Start free trial»)
2. Hero (заголовок, 2 CTA, анимация талона)
3. Сценарии применения — 4 иконки отраслей (без цифр)
4. Pillars — 3 карточки ценностного предложения
5. Как это работает — 3 шага
6. Features — 6 карточек функций
7. Interface Showcase — 4 скриншота
8. Use Cases — 6 отраслевых карточек
9. Pricing — динамически с API (статичный fallback)
10. FAQ — 8 вопросов
11. Footer CTA + юридическая полоса

## Рекомендации по улучшениям

### Приоритет 1 — Доверие и конверсия (быстрые победы)

#### 1.1 Редукторы риска рядом с CTA

**Проблема:** Кнопки «Start free trial» ничем не подкреплены — у пользователя нет ответов на скрытые возражения перед кликом.

**Решение:** Добавить строку из 3 иконок + текста под каждую primary CTA:
- «14 дней бесплатно» + «Без банковской карты» + «Отмена в любой момент»

**Реализация:** Добавить в `apps/marketing/components/landing/landing-hero.tsx` под блоком кнопок, и в `landing-footer-cta.tsx`. Текст уже ссылается на 14-дневный период — нужно только добавить визуальные значки (Lucide: `Calendar`, `CreditCard`, `X`).

#### 1.2 Продуктовые факты вместо иконок в Stats

**Проблема:** Секция «Common deployment scenarios» (`landing-stats.tsx`) показывает 4 иконки отраслей без каких-либо данных — визуальный шум без ценности.

**Решение:** Заменить на 4 факта о продукте — верифицируемые, честные, без клиентских метрик:

| Иконка | Факт |
|---|---|
| `Clock` | «Запуск за 2–3 часа» |
| `Plug` | «10+ готовых интеграций» |
| `Globe` | «2 языка интерфейса» |
| `CalendarDays` | «14 дней бесплатно» |

Это реальные характеристики продукта, не требующие клиентской базы для подтверждения.

#### 1.3 Trust-сигналы / безопасность

**Проблема:** Нет ни одного упоминания о безопасности данных, GDPR или хранении данных в России — это критично для B2B-аудитории, особенно в государственном секторе и здравоохранении.

**Решение:** Добавить полосу с иконками (можно встроить в Pillars или сделать отдельным блоком):
- «GDPR-совместимость»
- «Хранение данных в России» (ключевое для госсектора)
- «Соответствие 152-ФЗ»
- «SLA 99.9% для Enterprise»
- «Шифрование данных»

**Размещение:** Между Pricing и FAQ, или в виде полосы сразу под Hero.

#### 1.4 Строка логотипов интеграций

**Проблема:** Продукт поддерживает 10+ интеграций (Google Calendar, 1С, DaData, Twilio, SMS.ru, CalDAV, OIDC/SAML, Stripe, YooKassa) — ни одна не упомянута на лендинге. Это «скрытый» аргумент в пользу продукта.

**Решение:** Добавить `landing-integrations.tsx` — grayscale-полоса с иконками/логотипами:
```
Google Calendar | 1C | DaData | Twilio | SMS.ru | Stripe | YooKassa | SAML 2.0
```

**Размещение:** Под Hero или между Pillars и How It Works.

### Приоритет 2 — Социальное доказательство (отложено)

> Этот раздел не реализуется до появления реальных клиентов. Фиктивные цитаты, логотипы компаний и рейтинги без реальной базы подрывают доверие при первой же проверке и несут юридические риски (ЗоЗПП, FTC). Вместо этого — стратегия накопления.

#### 2.1 Стратегия получения первых отзывов

**Когда появятся 3–5 реальных пользователей:**
- Предложить им расширенный бесплатный период (30–60 дней) в обмен на честную обратную связь
- Запросить цитату: имя, должность, компания, что изменилось после внедрения
- Минимальный формат на сайте: 1 большая featured quote + 2 компактных карточки

**Компонент:** `apps/marketing/components/landing/landing-testimonials.tsx` — подготовить структуру заранее, но не добавлять в страницу до наличия реального контента.  
**Размещение (в будущем):** Между Use Cases и Pricing.

#### 2.2 Рейтинги на обзорных платформах

Зарегистрировать продукт на G2, Capterra или Product Hunt после первых клиентов. Бейджи добавлять только при наличии реальных оценок.

### Приоритет 3 — Контент и ценностное предложение

#### 3.1 Раздел «Book a Demo» с встроенным календарём

**Проблема:** Кнопка «Book a Demo» в Hero ведёт в никуда или на email — нет бесфрикционного способа записаться на демо. Корпоративные клиенты (Enterprise, госсектор) принципиально не принимают решения без живого разговора.

**Решение:**
- Добавить якорь `#book-demo` с отдельной секцией на лендинге
- Встроить Calendly/Cal.com виджет или кнопку `data-cal-link`
- Текст: «Мы покажем систему под ваши задачи — 30 минут, без обязательств»

**Размещение:** После Use Cases, перед Pricing. Альтернатива — отдельная страница `/demo`.

#### 3.2 Видео / product walkthrough

**Проблема:** Интерфейс показан 4 статичными скриншотами в `landing-interface-showcase.tsx` — сложно понять «как это работает» в динамике. Статичные изображения не передают real-time характер продукта.

**Решение:**
- Embed видео ≤2 минуты (YouTube/Vimeo): создание талона на киоске → появление в очереди → вызов на экране
- Или: интерактивный product tour через Arcade/Navattic (без кода)
- **Lite-вариант:** Добавить play-кнопку поверх скриншота в Interface Showcase → открывает видео в modal

#### 3.3 Секция «Старый способ vs. QuokkaQ»

**Проблема:** Нет явного контраста с status quo — непонятно, от чего именно уходит покупатель.

**Решение:** Сравнительная таблица в 2 колонки — «До» vs «После»:

| Традиционный подход | С QuokkaQ |
|---|---|
| Бумажные талоны и ручной учёт | Цифровой киоск, автоматический учёт |
| Нет данных о времени ожидания | Реальная аналитика, SLA-контроль |
| Клиент не знает, когда его вызовут | SMS/экран + живой ETA |
| Настройка занимает дни или недели | Запуск за 2–3 часа |
| Разные системы для разных филиалов | Единая платформа для всей сети |

**Компонент:** `apps/marketing/components/landing/landing-comparison.tsx` (новый)  
**Размещение:** После «Как это работает», перед Features.

#### 3.4 Sticky CTA на мобильных устройствах

**Проблема:** На мобайле пользователь скроллит длинную страницу без постоянно доступного способа начать пробный период.

**Решение:** Sticky bottom bar — только на мобильных (`lg:hidden`):
- Появляется после скролла за Hero (IntersectionObserver)
- Скрывается при открытии мобильного меню
- Кнопка: «Начать бесплатно» → ссылка на signup

**Компонент:** `apps/marketing/components/landing/landing-sticky-mobile-cta.tsx` (новый)

### Приоритет 4 — FAQ и политика

#### 4.1 Дополнительные вопросы в FAQ

**Текущий FAQ:** 8 вопросов — железо, уведомления, оффлайн, настройка, кастомизация, пробный период, пиковые часы, интеграции.

**Предлагаемые добавления** (в `apps/marketing/src/messages.ts`):

**Q9: «Соответствует ли продукт GDPR и 152-ФЗ?»**  
A: «Да — данные хранятся на серверах в России, полное соответствие 152-ФЗ. Для клиентов из ЕС возможен хостинг на EU-серверах с соответствием GDPR. Подробнее — в нашей Политике конфиденциальности.»

**Q10: «Можно ли экспортировать данные?»**  
A: «Да — аналитика, отчёты и история талонов доступны для экспорта в CSV и PDF в любое время. После завершения подписки данные доступны ещё 30 дней.»

**Q11: «Есть ли помощь с переходом с другой системы?»**  
A: «Для тарифов Professional и Enterprise — да. Наша команда помогает с импортом данных, настройкой и обучением сотрудников.»

**Q12: «Какие языки интерфейса поддерживаются?»**  
A: «Киоск и рабочие панели доступны на русском и английском языках. Дополнительные языки добавляются по запросу.»

#### 4.2 Исправление ценообразования в статичном fallback

**Проблема:** Fallback-цены в `apps/marketing/src/messages.ts` указаны в USD ($49, $149), тогда как реальные цены — в рублях (₽2 900, ₽9 900). Если API недоступен, пользователь видит заведомо неверные цифры.

**Решение:** Обновить fallback до реальных RUB-цен. Или — убрать конкретные суммы из статичного текста, заменив на «от ₽2 900/мес» без детализации по планам, чтобы fallback не вводил в заблуждение.

### Приоритет 5 — SEO и технические улучшения

#### 5.1 FAQPage JSON-LD Schema

**Проблема:** Structured data реализован для HomePage и PricingPage (`components/seo/`), но FAQ-секция не размечена — упущены rich snippets в Google SERP.

**Решение:** Добавить `FAQPage` JSON-LD в `home-page-json-ld.tsx` с вопросами из `messages.ts`. Отображается как раскрываемые Q&A прямо в результатах поиска — бесплатное расширение выдачи.

#### 5.2 Оптимизация изображений

**Проблема:** `public_screen.png` = 462 KB, `kiosk.png` = 273 KB. В `next.config.mjs` нет явной настройки — нужно убедиться что `unoptimized` отключён в production.

**Решение:**
- Конвертировать скриншоты в WebP/AVIF
- Добавить `blurDataURL` placeholder (устранит CLS при ленивой загрузке)
- Убедиться что Next.js Image Optimization включён в production-сборке

#### 5.3 OpenGraph preview

**Проблема:** Metadata ссылается на `/[locale]/opengraph-image` route — нужно убедиться что файл реализован и даёт привлекательный preview 1200×630 при расшаривании в Telegram/VK/LinkedIn.

#### 5.4 Core Web Vitals

**Рекомендация:** После добавления новых секций (video, integrations, comparison) — запустить Lighthouse аудит. Ключевые метрики:
- **LCP**: должен быть hero image (сейчас крупный PNG)
- **CLS**: динамическая загрузка цен из API может сдвигать layout
- **INP**: анимации на скролле

### Приоритет 6 — Долгосрочное развитие

#### 6.1 Блог / Ресурсный хаб

**Обоснование:** Без контентного маркетинга органический трафик будет крайне ограничен. Целевые ключевые запросы: «система управления очередью», «программа для электронной очереди», «CRM для очереди клиентов», «QMS для банка / МФЦ / клиники».

**Минимальный старт:**
- 3–5 статей: «Как выбрать QMS», «Электронная очередь в МФЦ», «ROI от внедрения системы очередей»
- Страница `/[locale]/blog` в `apps/marketing/app/`
- Ссылка в footer

#### 6.2 Exit Intent / Email-захват

**Проблема:** Пользователь уходит без конверсии — нет механизма дальнейшего nurture.

**Решение:**
- Exit intent модал: «Не готовы сейчас? Получите гид "Как выбрать QMS для вашей отрасли" на email»
- Или footer email-подписка: «Кейсы и новости раз в месяц»
- Интеграция с email-рассылкой (SendGrid / Unisender)

#### 6.3 ROI-калькулятор

**Обоснование:** B2B-клиенты (особенно корпоративные) должны обосновать покупку внутри компании — интерактивный расчёт экономии снижает барьер.

**Минимальный вариант** — 3 слайдера:
- Число посетителей в день
- Среднее время ожидания сейчас (мин)
- Число локаций / филиалов

→ Результат: «Экономия X часов работы персонала в месяц» или «Потенциальное снижение AHT на Y%»
