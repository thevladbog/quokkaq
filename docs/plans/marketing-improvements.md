# Лендинг (marketing): анализ и план улучшений

## Текущее состояние (актуально на 2026-04)

Главная [`apps/marketing/app/[locale]/page.tsx`](../../apps/marketing/app/[locale]/page.tsx) собирает цепочку:

1. **LandingTopBar** — лого, анкоры, CTA
2. **LandingHero** — заголовок, редукторы риска у CTA, демо
3. **LandingIntegrations** — полоса интеграций / протоколов
4. **LandingStats** — гибрид статичных фактов + опционально живые агрегаты из API
5. **LandingPillars** — три ценностных столпа
6. **LandingHowItWorks** — три шага
7. **LandingComparison** — «до / после»
8. **LandingFeatures** — шесть карточек
9. **LandingInterfaceShowcase** — скрины (WebP), опционально lite-video на первой карточке
10. **LandingUseCases** — отраслевые сценарии
11. **LandingBookDemo** — `#book-demo`, Cal.com iframe или fallback (lead / mailto)
12. **LandingPricing** — API + резерв высоты / fallback в RUB
13. **LandingTrustBadges** — GDPR / 152-ФЗ / хостинг и т.д.
14. **LandingFAQ** — расширенный FAQ + JSON-LD на главной
15. **LandingStickyMobileCta** — мобильный sticky CTA
16. **LandingFooterCta** — финальный CTA + legal + cookie settings

Дополнительные маршруты маркетинг-приложения: **`/{locale}/pricing`**, **`/{locale}/privacy`**, **`/{locale}/terms`**, **`/{locale}/blog`** (список и посты), **`/{locale}/roi`** (калькулятор оценки эффекта). Секция отзывов на главной подключается через компонент с **пустым массивом данных** — секция не рендерится, пока нет реальных кейсов.

## Анализ разрывов (остаточные и долгосрочные)

Легенда: **Готово** — внедрено в коде; **В работе / полировка** — есть база, нужны итерации; **Ожидает** — зависит от контента, юр. согласований или продукта.

| Элемент | Мировая практика | Статус |
|---|---|---|
| Социальное доказательство | Логотипы клиентов, цитаты, G2/Capterra | Ожидает первых клиентов; заготовка `landing-testimonials` без фейков |
| Видео / полноценный walkthrough | Embed ≤2 мин, продукт-тур | Полировка: env для embed; расширение контента |
| Карточки use cases с CTA | Ссылка на case study | Ожидает материалов |
| FAQPage JSON-LD | Rich results | Готово (`home-page-json-ld`) |
| Редукторы риска у CTA | «Без карты», триал | Готово |
| Stats с фактами / API | Честные метрики | Готово (гибрид) |
| Интеграции на лендинге | Полоса логотипов | Готово |
| Trust-сигналы | GDPR, 152-ФЗ, регион | Готово (копия согласовать с Legal при изменениях продукта) |
| Sticky CTA на мобайле | Нижняя панель | Готово |
| FAQ GDPR / экспорт / языки | Q9–Q12 | Готово |
| Сравнение подходов | До vs QuokkaQ | Готово |
| Book a Demo + Cal.com | Embed | Готово; при деплое с жёстким CSP см. README (`frame-src`) |
| Fallback цен RUB | Согласованность с API | Готово |
| OG preview | 1200×630 | В работе: кламп длины description в `opengraph-image` |
| WebP/AVIF, LCP | Оптимизация изображений | Готово для showcase; периодический аудит Lighthouse |
| Блог / ресурсы | SEO, nurture | Готово (инфра + стартовые посты); наполнение — итеративно |
| ROI-калькулятор | Оценка эффекта для B2B | Готово (`/roi`, дисклеймер) |
| Exit-intent | Повторный контакт | Готово (v1: без ESP; только при согласии analytics в cookie) |
| Exit-intent + ESP | Рассылка, CRM | Ожидает выбора ESP и юр. модели |

## Рекомендации по улучшениям (исторические разделы 1–6)

Ниже сохранена структура исходного документа с решениями по пунктам. Для **уже реализованного** см. таблицу выше и код в `apps/marketing`.

### Приоритет 1 — доверие и конверсия

Пункты 1.1 (редукторы), 1.2 (stats), 1.3 (trust), 1.4 (интеграции) — **реализованы** на главной.

### Приоритет 2 — социальное доказательство

Без фейковых цитат; компонент отзывов подключается при появлении `items`.

### Приоритет 3 — контент и ценностное предложение

3.1 Book a Demo / Cal.com, 3.3 comparison, 3.4 sticky CTA — **готово**.  
3.2 Видео — lite-modal по env `NEXT_PUBLIC_MARKETING_DEMO_VIDEO_EMBED`.

### Приоритет 4 — FAQ и политика

4.1–4.2 — **готово** (в т.ч. RUB fallback).

### Приоритет 5 — SEO и технические улучшения

5.1 JSON-LD, 5.2 изображения, 5.3 OG — см. код; периодически проверять Rich Results / превью соцсетей.  
5.4 CWV — прогон Lighthouse после крупных UI-изменений.

### Приоритет 6 — долгосрочное развитие

**6.1 Блог** — маршруты `/[locale]/blog`, контент в `content/blog/{locale}/*.md`, ссылка в футере, sitemap.  
**6.2 Exit intent** — модал на главной при согласии на аналитические cookies; первая итерация без стороннего ESP.  
**6.3 ROI** — страница `/[locale]/roi`, методология и дисклеймер в копии.
