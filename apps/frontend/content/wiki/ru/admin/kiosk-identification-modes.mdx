# Режимы идентификации на киоске

Для **конечной** услуги в киоске задаётся **режим идентификации** (в настройках услуги). Он определяет, что происходит **до** выдачи талона.

| Режим     | Действие посетителя                                      | Требования / заметки                                                                                                         |
| --------- | -------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| **none**  | Нет шага — талон сразу                                   | —                                                                                                                            |
| **phone** | Телефон (в т.ч. SMS, если включено)                      | Как в конфиге киоска/тарифа.                                                                                                 |
| **qr**    | Код **предзаписи/записи** (скан или ввод)                | Нужны **предзапись или чек-ин по записи** на подразделении. Если выключено, показывайте явное сообщение, а не пустой шаг.    |
| **login** | Табельный ввод; разбор на сервере через **employee IdP** | **IdP** включён у юнита (тариф + HTTPS). **Resolve** — только **киоск/терминал**; в **вебе** сотрудники ходят через **SSO**. |
| **badge** | **Бейдж** (wedge/serial)                                 | Тот же **IdP**, что и login; киоск шлёт строку на **resolve** на сервере.                                                    |

**SSO в браузере:** **веб**-приложение для персонала использует **SSO**. **Бейдж** и **resolve** IdP — для **киоска/терминала**, не для обычного сценария входа в веб-кабинет.

**Считыватели:** USB **HID** (эмуляция клавиатуры) и **последовательный** порт через десктоп-агент — см. `docs/operator/kiosk-scanner.md` в репозитории.

**HTTP-контракт к вашему API каталога** — в tenant-facing `developer-api` (OpenAPI к вашему API).
