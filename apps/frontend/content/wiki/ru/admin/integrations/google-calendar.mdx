# Google Календарь

Подключение **Google Календаря** нужно, чтобы QuokkaQ синхронизировал слоты предзаписи с календарём, которым вы управляете. В системе это хранится как **Google CalDAV** (`google_caldav`): используется CalDAV-совместимый доступ с учётными данными, полученными через OAuth.

## Где настраивать

1. Откройте **Настройки** → **Интеграции**.
2. Вкладка **«Календари»**.
3. **Добавить календарь** → **Google Календарь**.
4. Выберите **подразделение**, для которого подключается календарь, затем продолжите вход в Google.

Список подключений можно отфильтровать по подразделению селектором на той же странице.

## Как проходит подключение

1. Редирект на **Google**: вход и выдача прав (в том числе **офлайн-доступ**, чтобы сервер мог обновлять токен).
2. После согласия браузер возвращается в QuokkaQ с короткой сессией **выбора календаря**.
3. В диалоге показаны календари, куда у аккаунта есть право **записи** (владелец или редактор). Выберите календарь для этого подразделения и подтвердите.
4. Подключение появится в списке календарей выбранного подразделения.

Подключаться можно **не только к основному календарю**: доступен любой календарь с правом записи в этом аккаунте Google.

## Ограничения

- Не более **четырёх** подключений календаря **на одно подразделение** (Google и Яндекс вместе считаются в этот лимит).
- В списке выбора отображаются только календари с правом **записи**.

## Что должно быть настроено на сервере

Для сценария Google нужны **OAuth для Calendar** и доступный **Redis** (в Redis хранятся state OAuth и короткая сессия после входа, пока вы выбираете календарь).

Типичные переменные окружения (см. также `apps/backend/.env.example` в репозитории):

- `GOOGLE_CALENDAR_OAUTH_CLIENT_ID`
- `GOOGLE_CALENDAR_OAUTH_CLIENT_SECRET`
- `GOOGLE_CALENDAR_OAUTH_REDIRECT_URL` — должен совпадать с redirect URI в Google Cloud Console (origin API + путь `/calendar-integrations/google/oauth/callback`).

Если OAuth не настроен или Redis недоступен, при старте подключения или после возврата из Google отобразится ошибка.

## Типовые проблемы

| Симптом                                                  | Что сделать                                                                                      |
| -------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| Сообщение, что Google на сервере не настроен             | Попросить инфраструктуру выставить три переменные `GOOGLE_CALENDAR_OAUTH_*` и перезапустить API. |
| Нет refresh token от Google                              | Повторить подключение, пройти согласие заново, убедиться, что приложению выдан офлайн-доступ.    |
| «Сессия выбора календаря истекла» / invalid pick session | Шаг выбора действует ограниченное время (~15 минут). Начать **Подключить Google** снова.         |
| Лимит четырёх календарей у подразделения                 | Удалить неиспользуемое подключение или выбрать другое подразделение.                             |
| Ошибка после Google с текстом про Redis / `pick_save`    | API должен видеть Redis для этого сценария.                                                      |

## После подключения

URL CalDAV и идентификаторы доступа Google **фиксируются** при создании записи; в карточке интеграции по продукту можно менять, например, подпись, включение, часовой пояс (где поддерживается) и удалить подключение при необходимости.
