# Яндекс.Календарь (CalDAV)

Подключение **Яндекс.Календаря** через протокол **CalDAV**. В системе тип записи — **Yandex CalDAV** (`yandex_caldav`). Нужны URL календаря, логин Яндекса и **пароль приложения** (не обычный пароль от аккаунта).

## Где настраивать

1. **Настройки** → **Интеграции**.
2. Вкладка **«Календари»**.
3. **Добавить календарь** → **Яндекс.Календарь (CalDAV)**.
4. Выберите **подразделение**, заполните поля CalDAV и сохраните.

## Обязательные поля

- **Базовый URL CalDAV** — по умолчанию `https://caldav.yandex.ru`, если не используется другой endpoint.
- **Путь к календарю** — путь коллекции событий в интерфейсе Яндекса, обычно вида `/calendars/.../events/.../`.
- **Логин Яндекса** — аккаунт, которому принадлежит календарь.
- **Пароль приложения** — создаётся в настройках безопасности Яндекс ID для доступа к почте/CalDAV. Указывайте **его**, а не основной пароль.

## Часовой пояс

Укажите **IANA**-имя пояса (например `Europe/Moscow`), чтобы время слотов из календаря однозначно интерпретировалось в QuokkaQ.

## Ограничения

- Не более **четырёх** подключений календаря **на одно подразделение** (Google и Яндекс вместе учитываются в лимите).

## Безопасность

Пароли приложений на сервере **хранятся в зашифрованном виде** и не отдаются обратно в API после сохранения.

## Типовые проблемы

| Симптом                                      | Что сделать                                                                                                                           |
| -------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| Не сохраняется / требуется пароль приложения | При создании введите пароль приложения; при повторном включении отключённой интеграции без сохранённого секрета снова укажите пароль. |
| Неверный путь / ошибки синхронизации         | Проверьте путь CalDAV в Яндексе: нужна коллекция **событий** (`events`) нужного календаря.                                            |
| У подразделения уже четыре календаря         | Удалите лишнее подключение или выберите другое подразделение.                                                                         |

## После подключения

В карточке интеграции (где это поддерживает продукт) можно менять подпись, включение, часовой пояс, почты уведомлений и **обновить пароль приложения**.
