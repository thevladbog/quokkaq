# SMS-уведомления

QuokkaQ может автоматически отправлять посетителям SMS:

- **«Ваш талон вызван»** — когда оператор нажимает «Вызвать», «Перевызвать» или вручную берёт талон.
- **«Вы следующий»** — когда посетитель становится первым в очереди.

> **Требование к тарифу:** функция работает только при наличии в плане компании возможности `visitor_notifications`.

---

## Поддерживаемые провайдеры

| Провайдер   | Идентификатор | Особенности                                                                   |
| ----------- | ------------- | ----------------------------------------------------------------------------- |
| **SMSC**    | `smsc`        | Популярный российский шлюз; поддерживает буквенно-цифровые имена отправителей |
| **SMS.ru**  | `smsru`       | Российский сервис; API-ключ без пароля                                        |
| **SMSAero** | `smsaero`     | Российский сервис; авторизация по e-mail + API-ключ                           |
| **Twilio**  | `twilio`      | Международный провайдер; Account SID + Auth Token                             |

---

## Настройка в интерфейсе

1. Перейдите в **Платформа → Интеграции**.
2. Найдите раздел **SMS-уведомления**.
3. Выберите провайдера из выпадающего списка.
4. Введите учётные данные:
   - **API-ключ** — логин, Account SID или API ID в зависимости от провайдера.
   - **API-секрет** — пароль или Auth Token (для SMS.ru не требуется).
   - **Имя отправителя** — буквенно-цифровое имя, отображаемое в SMS (если провайдер поддерживает).
5. Установите переключатель **SMS включены**.
6. Нажмите **Сохранить**.

### Проверка настройки

После сохранения воспользуйтесь полем **Тестовое SMS** — введите номер телефона в формате E.164 (например, `+79001234567`) и нажмите **Отправить тест**. При успехе отобразится имя использованного провайдера.

---

## Переопределение через переменные окружения

Для production-развёртываний учётные данные рекомендуется передавать через переменные окружения, а не хранить в базе данных:

| Переменная       | Поведение                                       |
| ---------------- | ----------------------------------------------- |
| `SMS_API_KEY`    | Всегда переопределяет значение из БД            |
| `SMS_API_SECRET` | Всегда переопределяет значение из БД            |
| `SMS_PROVIDER`   | Заполняет провайдера, если поле в БД пусто      |
| `SMS_FROM_NAME`  | Заполняет имя отправителя, если поле в БД пусто |

> Если переменная `SMS_PROVIDER` установлена, SMS считается включённым даже при `sms_enabled = false` в БД.

---

## Поведение без настройки

Если SMS не настроено или отключено, система использует внутренний `log`-провайдер: сообщения выводятся в лог сервера и **не отправляются** реальным посетителям. Это безопасный режим по умолчанию для разработки и тестирования.

---

## Opt-in посетителя

Посетитель, получивший талон через киоск или виртуальную очередь, может добавить номер телефона прямо на странице талона (`/ticket/{id}`). Форма ввода отображается, когда:

- талон находится в статусе **ожидание**;
- SMS-уведомления включены и настроены (`smsOptInAvailable = true`).

После привязки телефона посетитель будет получать уведомления о вызове и позиции в очереди.
