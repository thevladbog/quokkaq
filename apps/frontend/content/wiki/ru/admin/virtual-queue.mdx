# Виртуальная очередь

Виртуальная очередь позволяет посетителям занять место в очереди **удалённо** — через QR-код или ссылку — не находясь физически в помещении. Они приходят, когда их вызывают или когда очередь доходит до них.

> **Требование к тарифу:** функция работает только при наличии в плане компании возможности `virtual_queue`.

---

## Включение для подразделения

1. Перейдите в **Настройки → Подразделения** и выберите нужное подразделение.
2. Откройте раздел **Виртуальная очередь**.
3. Включите переключатель **Виртуальная очередь активна**.
4. Сохраните настройки.

После этого на экране отображения подразделения появится QR-код, ведущий на страницу `/queue/{unitId}`.

---

## Как посетитель присоединяется к очереди

1. Посетитель сканирует QR-код на экране или переходит по ссылке.
2. На странице очереди отображается:
   - список доступных услуг;
   - текущая длина очереди и расчётное время ожидания.
3. Посетитель выбирает услугу и нажимает **Встать в очередь**.
4. Система создаёт талон и открывает страницу отслеживания (`/ticket/{id}`).

### Опциональная привязка телефона

На странице занятия очереди посетитель может указать номер телефона. После привязки он будет получать SMS:

- когда его талон вызван;
- когда он стал первым в очереди («Вы следующий»).

---

## Расчётное время ожидания (ETA)

Страница очереди и страница талона автоматически обновляют расчётное время ожидания каждые 30 секунд.

**Алгоритм:**

- берётся скользящее среднее времени обслуживания по последним 20 завершённым талонам;
- результат делится на количество активных окон;
- при недостатке данных (менее 3 завершённых) используется значение `MaxWaitingTime` из настроек услуги.

Когда в очереди больше одной услуги, на экране и в API также отображается разбивка по каждой услуге (`services[]` в ответе `/units/{unitId}/queue-status`).

---

## Отмена талона посетителем

Посетитель может самостоятельно отменить ожидающий талон прямо на странице `/ticket/{id}`:

1. Нажмите **Отменить талон**.
2. Статус изменится на `no_show`, история обновится действием `ticket.visitor_cancelled`.
3. На странице появится кнопка **Получить новый талон**, ведущая обратно на страницу очереди подразделения.

> Отмена возможна только для талонов в статусе **ожидание**.

---

## Публичный API статуса очереди

Эндпоинт `GET /units/{unitId}/queue-status` доступен без авторизации и возвращает:

```json
{
  "queueLength": 5,
  "estimatedWaitMinutes": 12.5,
  "activeCounters": 2,
  "services": [
    {
      "serviceId": "...",
      "serviceName": "Консультация",
      "queueLength": 3,
      "estimatedWaitMinutes": 7.5
    },
    {
      "serviceId": "...",
      "serviceName": "Документы",
      "queueLength": 2,
      "estimatedWaitMinutes": 5.0
    }
  ]
}
```

Поле `services` присутствует только при наличии ожидающих талонов более чем по одной услуге.
